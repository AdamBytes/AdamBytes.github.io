HTB: Explosion

Date: 17 October 2025

Lab Objective: ‘Explosion’ is the fifth machine in the ‘Starting Point - Tier 0’ lab series by Hack the Box. Explosion specifically sits with understanding penetration fundamentals - especially if they are misconfigured.

In this case, we are looking at the Remote Desktop Protocol Specifically. By taking advantage of a misconfigured system, a threat actor can fully compromise the entire system.

Important Considerations

RDP: Remote Desktop Protocol - allows for team members to remotely connect to other hosts.

Ports: 3389 TCP and 3389 UDP.

Telnet: Outdated tool, often unsecure. Left for SSH.

SSH: Secure Shell Protocol. Added layers of authentication and encryption to the base communication model.

Use case: patch delivery, file transfers, log transfers and remote management.

Enumeration

As always, we need to start with enumeration (gathering information) as it is the first and most critical step in penetration testing. This would involve gathering information about the target system by slowly mapping out key vulnerabilities - it’s making an assessment. Skipping this step can lead to missed vulnerabilities and wasted time.

We will start with using Nmap: nmap -p -sV {targetIP}. When we use -p- we are scanning all 65535 TCP ports and with -sV we are detecting service versions.

Task 1 - What does the 3-letter acronym RDP stand for?

Answer: Remote Desktop Protocol.

Task 2 - What is the 3-letter acronym that refers to interaction with the host through the command line interface?

Answer: CLI

Answer: GUI

Answer: Telnet
