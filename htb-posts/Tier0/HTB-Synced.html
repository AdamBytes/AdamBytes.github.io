HTB: Synced

Date: 13th September 2025

Lab Objective: ‘Synced’ is a Tier 0 machine in the Starting Point series. This lab focuses on the Rsync protocol. Rsync (Remote Sync) is a utility for efficiently transferring and synchronizing files between a computer and a storage drive and across networked computers by comparing the modification times and sizes of files.

Important Considerations

Port: Port 873 TCP is the default port used by Rsync. It can be used for local and remote file-copying.

Configuration: Rsync uses "modules" which are essentially directory shares. These can be configured with or without authentication. In a professional environment, allowing anonymous access to these modules can lead to significant data leaks.

Security: Similar to the Telnet issues found in Meow, Rsync does not provide encryption by default during transport unless it is tunneled through a protocol like SSH. If misconfigured, an attacker can list, read, and sometimes write files to the server without a password.

Enumeration

As always, we need to start with enumeration. This involves gathering information about the target system to identify what services are exposed. Skipping this step often leads to rabbit holes and wasted time.

Task 1 - What is the default port for rsync?

Answer: 873

sudo nmap -sV [TARGET_IP]

Task 2 - How many TCP ports are open on the remote host?

Answer: 1

Task 3 - What is the protocol version used by rsync on the remote host?

Answer: 31

Task 4 - What is the command used to list the modules available on the target?

Answer: rsync --list-only

To interact with this service, ensure you have the rsync client installed on your local machine. You can install it using: sudo apt install rsync.

Establishing a foothold

Task 5: What is the name of the share (module) that is available on the target?

Answer: public

I used the command rsync --list-only [TARGET_IP]:: to see what was available. The double colon :: is used to contact the rsync daemon directly.

Task 6: What is the command to list the contents of the 'public' share?

Answer: rsync [TARGET_IP]::public

Task 7: What is the name of the file found inside the share?

Answer: flag.txt

Once the file was identified, I needed to "sync" it to my local machine to read it. I used the command:

rsync [TARGET_IP]::public/flag.txt .

The . indicates that I want the file copied to my current working directory.

Task 8: Submit the Flag

Answer: [The hash found inside flag.txt]

Final Thoughts

  • Rsync is a powerful tool, but dangerous when the "Anonymous" login is left enabled on sensitive directories.
  • The --list-only flag is your best friend when enumerating unknown rsync targets.
  • Always check for "modules" before trying to guess file paths.
