HTB: Preignition

Date: 17th October 2025

Lab Objective: ‘Preignition’ is the sixth machine in the ‘Starting Point - Tier 0’ lab series by Hack the Box. This box focuses on fundamental web enumeration and the discovery of administrative interfaces through directory brute-forcing. It's an introduction to tools like gobuster and the concept of "low-hanging fruit" such as default login credentials.

Enumeration

As always, we need to start with enumeration (gathering information) as it is the first and most critical step in penetration testing. This would involve gathering information about the target system by slowly mapping out key vulnerabilities - it’s making an assessment. Skipping this step can lead to missed vulnerabilities and wasted time. In this case, we start by using Nmap.

We start by scanning the target IP address to identify open ports and services: nmap -sV {target_IP}

The scan reveals a single open port: port 80 (HTTP) and it is running nginx 1.14.2

We then proceed to navigate to http://{target_IP} in the browser and it shows a standard Nginx "Welcome" page. There are no visible links or clues on the landing page, so we need to look for hidden files or directories.

To find hidden paths, we use gobuster. This was my first introduction to the tool and had to use the walkthrough to better understand what was required. Overall, the general principle is to use a common wordlist and look specifically for .php files or other webfiles not listed, as this is a common extension for administrative portals : gobuster dir -u http://10.129.20.25 -w /usr/share/wordlists/dirb/common.txt -x php

We are given the result: /admin.php (Status: 200), succesfully identifying it as a login page

To access the admin panel we navigate to http://{target_IP}/admin.php and that brings up a simple login form requesting username and password.

Before trying complex exploits, it is standard practice to test for default credentials. Often, "admin/admin" is left as the default configuration.

  • Username: admin
  • Password: admin

Once logged in, the page refreshes to show the internal dashboard. The flag is displayed clearly on the screen.

Final Thoughts

  • This lab shows that you don't always need a fancy exploit to get in. Simply looking for hidden folders (fuzzing) and trying common passwords like "admin" is often enough to find a way inside a system.
  • Just because a login page isn't linked on the main website doesn't mean it's safe. True security comes from using strong, unique passwords, not just hoping that no one finds your "secret" admin page.
