HTB: Redeemer

Date: 11 October 2025

Lab Objective: ‘Redeemer’ is the fourth machine in the Start Point – Tier 0’ series on HTB. After completing the lab and spending some time researching, it became evident that this lab is to find the flag by exploiting a misconfigured Redis database server.

Important Considerations

  • Redis is an ‘In-Memory Database’. This database is volatile, stored as simple key-value pairs and is designed for caching.

  • The key tool that we are using is a dedicated client for this service; redis-cli

  • The flag will be stored as a value inside a key

Enumeration

As always, we need to start with enumeration (gathering information) as it is the first and most critical step in penetration testing. This would involve gathering information about the target system by slowly mapping out key vulnerabilities - it’s making an assessment. Skipping this step can lead to missed vulnerabilities and wasted time.

We will start with using Nmap: nmap -p -sV {targetIP}. When we use -p- we are scanning all 65535 TCP ports and with -sV we are detecting service versions.

Task 1 - Which TCP port is open on the machine?

Answer: 6379

sudo nmap -sV

Task 2 - Which service is running on the port that is open on the machine?

Answer: redis

Redis is a REmote DIctionary Server that is an open-source advanced NoSQL key-value data store used as a database, cache and message broker. It is used for short term storage for fast retrieval.

Task 3 - What type of database is Redis? Choose from the following options: (i) In-memory Database, (ii) Traditional Database

Answer: In-memory Database

Redis runs server side software - the server listens for connections from clients, with a GUI or through the CLI.

The CLI is a tool that gives complete access and its functionalities.

The database is stored in the server’s RAM to enable fast data access. To interact with the server we need to install sudo apt install redis-tools

Task 4: Which command-line utility is used to interact with the Redis server? Enter the program name you would enter into the terminal without any arguments.

Answer: redis-cli

Establishing a foothold

Task 5: Which flag is used with the Redis command-line utility to specify the hostname?

Answer: -h

Task 6: Once connected to a Redis server, which command is used to obtain the information and statistics about the Redis server?

Answer: info

Task 7:What is the version of the Redis server being used on the target machine?

Answer: 5.0.7

Task 8: Which command is used to select the desired database in Redis?

Answer: select

Task 9: How many keys are present inside the database with index 0?

Answer: 4

Task 10: Which command is used to obtain all the keys in a database?

keys *

Use the command get on each of the stored keys, and one of them will have a flag.

Answer: follow the instructions

Final Thoughts

  • If a complex service is exposed, prioritize using its native client over generic tools
  • In a key-value store, the flags or critical data are often named obviously (flag, password, secret). The exploitation path is simply discovery, selection, and retrieval.
  • The primary security lesson is that sensitive data services must never be accessible without authentication, even if they are only running on a non-standard port..
