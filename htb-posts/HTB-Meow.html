HTB: Meow

Date: 13 September 2025

Meow is the beginning of the Starting Point process on Hack The Box labs. The learning outcomes for the Starting Point program are to gain fundamental skills for penetration testing. In these tutorials, we learn to connect to various services such as FTP, SFTP, SMB, Telnet, RSync, and RDP anonymously.

We also learn tools such as Nmap, the Bash terminal, and how to communicate with machines and identify vulnerabilities.

Enumeration

As a core process for any kind of penetration testing, we need to spend time gathering information from a client or target system. This is the first step and can save us a lot of time and hassle, so we need to make sure we don't overlook anything crucial.

Start Process

From my experience working with computers at home and in the workplace, I knew that Ping was the tool to check if the target host was reachable. I tried ping [TARGET IP] and confirmed it was returning responses. Though I had little experience with Linux (in this case, Parrot OS), I didn’t know how to ping a specific number of times or how to cancel it. After a quick search, I found that Ctrl + C cancels the ping, and -c specifies the count.

Nmap Scans

I had read about Nmap in HTB materials and my CompTIA coursework. In short, Nmap is a network mapper used for scanning and host discovery. It supports features like network discovery, port scanning, OS fingerprinting, vulnerability assessment, and network monitoring.

I learned two useful commands:

  • sudo nmap -sV [IP Address]: Scans for open ports and attempts to identify service versions.
  • sudo nmap -Pn [IP Address]: Skips host discovery and assumes the host is up. Useful when ICMP is blocked by firewalls.

From scanning the IP address, I discovered that a Telnet service was open on port 23.

Telnet: Connecting to the Box

Telnet is a protocol used to connect to remote machines via a command line interface. The command is telnet [IP address]. It then prompts for a username and password. Knowing Telnet is insecure, I tried common usernames like admin, administrator, and root—and using the same for the password. It let me in.

After researching insecure credentials, I learned that brute force attacks can be used with scripts or tools to guess login combinations. I also explored whether encoding Telnet with base64 would improve security—but it doesn’t. Telnet transmits data in plain text, meaning anyone can intercept and read it. Base64 is just encoding, not encryption, and can be easily decoded.

Once inside Telnet, I familiarised myself with its options using help and ?. I used ls to list files in the directory and discovered these useful commands:

  • ls -l: Displays files in long list format.
  • ls -a: Includes hidden files.

I noticed a file named flag.txt. I used cat flag.txt to read it, and it revealed the hash value needed to complete the room.

Final Thoughts

This was my first interaction with penetration testing, and it was exciting to learn the basics. I walked away with a better understanding of:

  • How to identify and connect to services
  • The importance of enumeration
  • Basic Linux navigation

It’s a simple start, but it lays the groundwork for more complex challenges ahead.
