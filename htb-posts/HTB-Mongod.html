HTB: Mongod

Date: 5 January 2026

Lab Objective: ‘Mongod’ is the seventh machine in the Start Point – Tier 0’ series on HTB that comes with VIP access.

In this case, we explore MongoDB. MongoDB is a document-oriented NoSQL database and has the following hierarchy: databases; collections and documents. For simplicity sake, databases are organised into collections which contain documents. These documents contain data in the format of strings, numbers, dates in a JSON-like format.

Important Considerations

I will make it very apparent that I used the Walkthrough extensively throughout this lab as I had no prior knowledge of MongoDB nor its system access level and the kind of exploits that could take place. I detail my observation and self-learning process under final thoughts at the end of this write-up.

What is important to understand is that misconfiguration typically leads to vulnerabilities in most environments. In this case, we are taking advantage of a misconfigured system to gain an anonymous login and then retrieve the respective flag.

I learnt that the ‘Mongosh’ command line utility was very important here. Below are the steps to enumerate and then take a foothold in the server.

Enumeration

As always, we need to start with enumeration (gathering information) as it is the first and most critical step in penetration testing. This would involve gathering information about the target system by slowly mapping out key vulnerabilities - it’s making an assessment. Skipping this step can lead to missed vulnerabilities and wasted time. In this case, we start by using Nmap.

With the help of the walkthrough, it was found that we had to use nmap -p- --min-rate-1000 -sV {TARGET-IP}

We add the extra conditionals to the command to make sure that it scans ports 0-65535; determines the versions of the running services and the minimum number of packets that Nmap should send per second.
